⚠️ Catatan: POC asli (Chromium Issue #1) memerlukan dua tab terpisah dengan mode Split View dan protokol file://.
Karena Anda minta digabung menjadi satu file HTML, kode di bawah ini mensimulasikan UI spoofing dengan iframe + popup window.
Untuk hasil maksimal, buka file ini di Chrome Canary dengan flag #side-by-side diaktifkan, lalu ikuti instruksi manual.

📁 Tab 1 (Attacker) – spoofpermisx.html

Klik kanan pada tautan di bawah, lalu pilih "Open link in split view" (jika mode split view aktif).

🔗 Open link in split view (klik kanan)

* Jika split view tidak aktif, tautan akan terbuka di tab baru biasa – tetap bisa diuji tapi efek spoofing kurang kuat.

Kode asli (dari laporan):

<script>
function triggerPermission() {
    navigator.mediaDevices.getUserMedia({ video: true })
        .then(() => console.log("izin diberikan"))
        .catch(e => console.log("ditolak"));
}
</script>

📄 Tab 2 (Korban) – perm.html

Prompt izin dari Tab 1 akan muncul di atas area ini seolah-olah berasal dari Tab 2.

* Dalam POC asli, perm.html hanya berisi teks "Click Allow" dan tidak perlu meminta izin sendiri.
