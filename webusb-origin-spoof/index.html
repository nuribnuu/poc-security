PoC: Origin Spoofing via about:blank Popup

This PoC demonstrates that permission dialogs for WebUSB, Web Serial, and WebHID do not show the correct origin when triggered from an about:blank popup window.

Expected secure behavior: The permission dialog should clearly display the requesting origin (e.g., https://your-domain.com).

Observed vulnerable behavior: The dialog shows only "wants to access the following permissions:" without any origin.

Test instructions:
1. Click the button above to open an about:blank popup.
2. In the popup, click any of the "Wrong (USB)", "Wrong (Serial)", or "Wrong (HID)" links.
3. Observe the permission dialog – the origin should be missing.
4. Compare with "Normal (Audio)" or "Normal (GeoCode)" – those correctly show the origin (because they use different permission APIs).
